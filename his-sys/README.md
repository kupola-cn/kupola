```
│ 分机构管理层  │ 本机构全部数据                    │
│ 分机构普通员工│ 本机构 + 授权范围内的数据          │
└──────────────┴──────────────────────────────────┘
```

### 四、权限粒度建议

建议采用 **四层权限粒度**：

| 层级 | 控制内容 | 示例 |
|------|----------|------|
| **1. 菜单权限** | 能看到哪些页面 | 分机构能看到"耗材管理"，集团看不到"出库操作" |
| **2. 操作权限** | 能执行哪些按钮/动作 | 库管员能"确认出库"，普通护士只能"查看库存" |
| **3. 数据权限** | 能看到哪些数据范围 | 集团看全部机构，分机构只看自己的 |
| **4. 字段权限** | 能看到哪些字段 | 普通员工看不到采购价格，管理层可见 |

### 五、耗材管理权限示例

```
集团（采购部）
├── 耗材目录管理（增删改）        ← 集团独占
├── 供应商管理                    ← 集团独占
├── 采购订单（下单、付款）        ← 集团独占
├── 查看各分机构库存/使用记录     ← 集团只读
│
分机构（库管员/护士）
├── 查看集团下发的耗材目录        ← 只读
├── 出库操作（领用、发放）        ← 分机构自主
├── 使用记录（患者耗材登记）      ← 分机构自主
├── 库存盘点                      ← 分机构自主
├── 申购申请（向集团申请采购）    ← 分机构发起
```

### 六、数据库核心表设计建议

| 表名 | 用途 |
|------|------|
| `organizations` | 组织表（集团+分机构统一存储，`parent_id` 区分层级） |
| `departments` | 部门表（`org_id` 关联到具体组织） |
| `roles` | 角色表（`org_id` 关联，集团和分机构各自定义） |
| `users` | 用户表（`org_id` 归属组织，`org_type` 区分集团/分机构） |
| `permissions` | 权限定义表（菜单+操作+数据范围） |
| `role_permissions` | 角色-权限关联 |
| `user_roles` | 用户-角色关联 |

### 七、确认的点

1. **字段权限**是否需要？（如采购价格对分机构不可见）— 这会增加复杂度: 不需要
2. **分机构之间**是否需要看到彼此的耗材库存？（如 A 中心看到 B 中心的库存）：不能
3. **耗材目录**是集团统一定义，分机构不能自定义？还是分机构可以添加自己的耗材？：统一定义
4. **角色模板**：集团是否预设一套"标准角色"供分机构选用，还是完全自由定义？：预设标准角色
