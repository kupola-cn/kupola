## 一、README.md —— 让用户“30 秒决定用不用你”

**核心使命**：用户打开 GitHub 的第一屏，立刻回答 3 个问题：**这是什么？我为什么用你？我最快怎么开始？**

**必须写的内容**（按顺序，7 块）：

1. **One-liner 定位**（1 句话）
   - 格式：`[库名] 是一个[核心特性]的[类型]，适用于[场景]`
   - 示例：`MyLib 是一个零框架依赖的声明式 UI 引擎 + 组件库，适用于任何服务端渲染 Web 应用。`
   - ⚠️ 不要写“高性能、轻量级、易用”——这些是形容词，不是定位。

2. **核心特性**（3-5 个 bullet points，每个带 emoji）
   - 示例：
     - 🪶 **零依赖**：核心库 < 8KB gzip
     - ⚛️ **声明式**：基于模板字面量，纯 JS 即用
     - 🖥️ **SSR 就绪**：renderToString + 纯净水合，无冲突
     - 🧩 **组件可选**：只用核心，或引入 Dialog/Tabs/Dropdown

3. **最快上手**（**第一段必须是 CDN `<script>`**）
   ```html
   <script src="https://cdn.jsdelivr.net/npm/my-lib@1.9.17/dist/browser.js"></script>
   <script>
     const { createApp, reactive } = MyLib;
     const state = reactive({ count: 0 });
     createApp(() => `<button onclick="state.count++">${state.count}</button>`).mount('#app');
   </script>
   ```
   - ⚠️ 这是你最重要的 10 行代码，用户复制就能跑。

4. **安装方式**（第二段，压缩到 3 行）
   ```bash
   npm install my-lib          # 完整版
   npm install @my-lib/core    # 仅核心引擎
   # 或 CDN: https://cdn.jsdelivr.net/npm/my-lib@1.9.17/dist/browser.js
   ```

5. **文档链接**（指向你部署的完整文档网站，如果有）
   - `📖 [完整文档](https://my-lib.dev)`
   - `📦 [API 参考](https://my-lib.dev/api)`

6. **生态/组件预览**（可选，但强烈推荐）
   - 放 3-5 个组件的截图或 GIF（Dialog、Tabs、Dropdown、Button、Toast）
   - 让用户一眼看到“这库能做出什么”

7. **Badge 行**（放在最顶部）
   - `![npm](https://img.shields.io/npm/v/my-lib)`
   - `![bundlephobia](https://img.shields.io/bundlephobia/minzip/my-lib)`
   - `![Build Status]`
   - `![License]`

**不需要写的内容**：
- ❌ 详细的 API 文档（放单独文档网站或 `docs/` 目录）
- ❌ 贡献指南（放 `CONTRIBUTING.md`）
- ❌ 集成指南（放 `INTEGRATION.md`）
- ❌ 版本历史（放 `CHANGELOG.md`）
- ❌ 作者的开发故事（放博客，不放 README）

**篇幅建议**：**200-400 行**（含空行和代码块）。太短不信任，太长没人读。

## 二、CONTRIBUTING.md —— 让贡献者“5 分钟内能跑起来”

**核心使命**：告诉想提 PR/Issue 的人：**我该怎么改？你的代码规范是什么？我提交前要过哪些检查？**

**必须写的内容**（9 块）：

1. **欢迎语**（1 段）
   > 感谢你对 MyLib 的兴趣！我们欢迎所有形式的贡献，包括 Issue、PR、文档改进和示例补充。

2. **行为准则**（1 行链接）
   > 请遵守 [Code of Conduct](./CODE_OF_CONDUCT.md)。

3. **开发环境准备**
   ```bash
   node >= 18.x
   npm >= 9.x
   ```

4. **一键启动**（从 0 到能跑的核心步骤，不超过 10 行）
   ```bash
   git clone https://github.com/yourname/my-lib.git
   cd my-lib
   npm install
   npm run build
   npm run dev   # 启动开发服务器
   ```

5. **项目结构**（树形图，只列一级目录）
   ```
   packages/
   ├── core/        # 核心引擎
   ├── components/  # UI 组件库
   └── main/        # 完整入口
   docs/             # 文档源码
   examples/         # 示例代码
   ```

6. **代码规范**（自动化的就写一行命令，手动的才列规则）
   ```bash
   npm run lint      # ESLint + Prettier
   npm run test      # 单元测试
   npm run typecheck # TypeScript 检查
   ```
   - 手动规则（只写 3-5 条最重要的）：
     - 使用 `const`/`let`，不用 `var`
     - 禁止 `console.log` 提交到生产（dev 除外）
     - 公开 API 必须有 JSDoc 或 TS 类型

7. **提交 PR 前检查清单**（可勾选框）
   - [ ] 代码通过所有 CI 检查
   - [ ] 新增功能包含测试用例
   - [ ] 更新了相关文档
   - [ ] `npm run build` 后体积未增加超过 500B
   - [ ] 已在 Chrome/Firefox/Safari 最新版手动测试

8. **Issue 提交规范**（模板链接）
   > Bug 报告请使用 [Issue Template](./.github/ISSUE_TEMPLATE/bug_report.md)，并附上最小复现仓库。

9. **联系方式**（可选）
   > 如有疑问，可在 [Discussions](https://github.com/yourname/my-lib/discussions) 提问。

**不需要写的内容**：
- ❌ 详细的设计理念（放 `docs/design.md`）
- ❌ 完整的 API 文档（放文档网站）
- ❌ 教程（放 `examples/` 或 `docs/guides/`）
- ❌ 如何集成到项目（放 `INTEGRATION.md`）

**篇幅建议**：**100-200 行**。贡献者只想快速上手，不想读长篇小说。

## 三、INTEGRATION.md —— 让用户“跟自己的技术栈对接”

**核心使命**：告诉已经决定用你的用户：**怎么在我的项目里用？怎么跟不同框架/环境配合？**

**必须写的内容**（按场景分块，不是按功能分块）：

### 1. 无构建项目（CDN）
```html
<script src="https://cdn.jsdelivr.net/npm/my-lib@1.9.17/dist/browser.js"></script>
<script>
  // 全局变量 MyLib
  const app = MyLib.createApp(...);
</script>
```

### 2. Node.js / 后端渲染（SSR）
```javascript
// 服务端
import { renderToString } from '@my-lib/core/server';

const html = renderToString(() => `<h1>Hello</h1>`);
// 输出纯 HTML
```

### 3. Vite / Webpack 工程化项目
```bash
npm install my-lib
```
```javascript
import { createApp, reactive } from 'my-lib';
// 或按需加载组件
import { Dialog } from 'my-lib/components/dialog';
```

### 4. 与 React 混用（如果需要）
```jsx
import { Dialog } from 'my-lib/components/dialog';

function MyReactPage() {
  useEffect(() => {
    const dialog = Dialog.open({ title: 'Hello' });
    return () => dialog.close(); // 清理
  }, []);
  return <div>...</div>;
}
```

### 5. 与 Vue 混用（如果需要）
```vue
<script setup>
import { onMounted, onUnmounted } from 'vue';
import { Dialog } from 'my-lib/components/dialog';

onMounted(() => {
  const dialog = Dialog.open({ title: 'Hello' });
  onUnmounted(() => dialog.close());
});
</script>
```

### 6. 常见集成问题（FAQ 风格，3-5 条）

- **Q：我的项目用了 Tailwind，会和你的 CSS 冲突吗？**
  - A：不会。所有组件样式都限制在 `.my-lib-*` 前缀内，且仅使用 CSS 变量，不依赖外部样式。

- **Q：服务端渲染时，`window is not defined` 报错怎么办？**
  - A：核心库的 `renderToString` 不依赖任何浏览器 API，可直接在 Node.js 中运行。

- **Q：我只需要 Dialog，不想引入整个组件库？**
  - A：`import { Dialog } from 'my-lib/components/dialog'` 即可按需加载。

**不需要写的内容**：
- ❌ 为什么选 MyLib（放 README）
- ❌ API 详细参数（放文档网站或 `docs/api/`）
- ❌ 贡献指南（放 CONTRIBUTING.md）
- ❌ 开发环境搭建（放 CONTRIBUTING.md）

**篇幅建议**：**80-150 行**。用户带着具体问题来，快速找到对应的那一节。

## 六、最后一句建议
